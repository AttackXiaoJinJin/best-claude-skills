# SKILL.md 骨架模板

复制以下内容到新 skill 的 `SKILL.md`，按注释提示填写，删除所有注释行。

---

```markdown
---
name: <skill-name>
description: >
  [触发场景：用户说/做什么时加载此 skill。用确定性语句，避免"可以"、"也许"。]
  [可选：列举更多触发词或场景。]
  [明确写出不触发的情况，防止误触发。]

  <!-- 示例：
  当用户要求创建数据库迁移、修改 schema、运行 migration 时调用。
  用户说"跑迁移"、"加字段"、"改表结构"时触发。
  普通 SQL 查询编写不触发。
  -->
---

# <Skill 名称>

[一句话说明：skill 解决什么问题，以及为什么非显而易见（Claude 默认行为哪里不够用）。]
```

## 各段落说明

| 段落 | 必填 | 目的 |
|------|------|------|
| Frontmatter (`name` + `description`) | 必填 | `description` 是触发条件，模型靠它决定是否加载 |
| 开头一句话 | 必填 | 快速定位 skill 价值，让 Claude 知道"为什么要用这个" |
| 前置条件 | 建议填 | 防止 Claude 在条件不满足时盲目执行 |
| 流程 | 必填 | 核心指令；给目标和约束，不要给操作列表 |
| Gotchas | 建议填 | 价值密度最高；暂无时明确写出，不要省略段落 |
| 配置 | 按需 | 有持久化配置需求时才写 |
| 参考 | 按需 | 有子文件时必须写，否则 Claude 不会主动读取子文件 |
| 验证 | 可选 | 记录对照测试结论；新 skill 建议填写；维护时用于快速确认 skill 仍有效 |
